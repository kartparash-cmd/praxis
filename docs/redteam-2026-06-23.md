## The one cross-cutting theme

**Both lists converge on the same fact: praxis owns no distribution surface and no
moat beyond its repo-anchored audit output.** Every fatal attack exploits the one
deliberate choice — a moat-less guest with git-clone distribution. Every
high-leverage absence (#2 packaging, #4 famous-repo audit, #5 versioning, #10
README) is that same gap from the defender's side. For a free forkable skill,
"moat" = (a) being the demonstrably best audit-led learning prompt and (b) Kay's
distribution and reputation around it. That ground is currently unclaimed.

## Defender's 3 moves (leverage order)

1. **Seed the shareable-audit channel before anyone floods it.** Run AUDIT on 3-5
   recognizable OSS repos, commit the artifacts (#4), post with receipts (reply to
   the 3 original authors, submit to Awesome Claude Code). The one moat praxis can
   build; blunts kill #4 by getting there first. Honor the 30-day kill date.
2. **Ship the credibility chain.** Tests+CI (#1) → VERSION/tag (#5) →
   `.claude-plugin` packaging (#2) → fix demo GIF + badges (#10). Converts "raw
   clone of an untested script" into "versioned, tested, one-click plugin" — the
   floor to survive comparison with a host-native command on anything but reach.
3. **Close the retention loop + double down on the repo-anchored half.** Add
   `praxis stats` + due-forecast (#3) and misconception capture (#9) so it scores
   the loop it optimizes; lean the framing entirely on *audit-YOUR-repo-for-
   evidence-backed-gaps* — the half topic-deck tools structurally cannot do.
